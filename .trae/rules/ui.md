# 非遗建筑APP设计规约

## 文档概述
本规约文档为非遗建筑主题移动应用的设计指导规范，涵盖视觉设计、交互逻辑、内容架构等核心要素，确保应用在传承非遗文化的同时提供优秀的用户体验。

## 1. 设计原则

### 1.1 文化传承原则
- 尊重传统建筑的文化内涵与历史背景
- 避免过度现代化的设计元素
- 保持建筑本体的真实呈现

### 1.2 用户体验原则
- 简洁直观的导航结构，遵循小程序原生导航规范
- 高可读性的文字排版
- 符合移动端和小程序特定的操作习惯

### 1.3 技术实现原则
- 响应式设计，使用 `rpx` (Responsive Pixel) 替代传统的 `px` 进行多端屏幕适配（规定 1px = 2rpx，设计稿按 750px 宽度出图）。
- **强制 uvue 纯 Flex 布局**：全面拥抱原生 Flex 渲染引擎，抛弃传统 Web 的 `float` 等布局方式。
- 考虑多端性能限制，避免过度复杂的动画和深层 DOM 结构，长列表优先使用 `<list-view>`。
- 遵循原生 App 与小程序的交互设计规范。
- 全面适配全面屏安全区（如通过 API 获取安全区或使用 `env(safe-area-inset-bottom)`）。

## 2. 视觉设计规范

### 2.1 配色方案

#### 核心色盘

| 色名 | HEX | RGB | 应用场景 |
| :--- | :--- | :--- | :--- |
| 梁木褐 | #7A5C48 | 122, 92, 72 | 主色调，用于导航栏、重要按钮 |
| 青砖灰 | #9E9E9E | 158, 158, 158 | 次要元素，背景分隔线 |
| 檐影青 | #4A6B75 | 74, 107, 117 | 强调色，用于选中状态 |
| 窗棂朱 | #C44536 | 196, 69, 54 | 交互色，按钮悬停、重要提示 |
| 素墙白 | #F8F5F0 | 248, 245, 240 | 背景色，页面主背景 |
| 瓦当黛 | #3D3D3D | 61, 61, 61 | 标题文字，重要内容 |
| 苔痕绿 | #8A9A8D | 138, 154, 141 | 辅助色，点缀元素 |

#### 色彩应用规范
- **背景层**：素墙白（#F8F5F0）
- **文字主体**：瓦当黛（#3D3D3D）
- **导航选中**：檐影青（#4A6B75）+ 窗棂朱指示条
- **交互按钮**：梁木褐为主，窗棂朱为悬停状态

### 2.2 字体规范

#### 中文字体
- **标题**：思源宋体 Medium (36-48rpx)
- **正文**：思源黑体 Regular (28-32rpx)
- **说明文字**：思源黑体 Light (24-28rpx)

#### 英文字体
- **标题**：Georgia Medium
- **正文**：Helvetica Neue Regular
- **代码**：Courier New

#### 字号层级 (以 750rpx 设计稿为准)
- **H1**: 48rpx - 页面标题
- **H2**: 40rpx - 模块标题
- **H3**: 36rpx - 小标题
- **P**: 32rpx - 正文内容
- **Caption**: 24rpx - 图片说明/辅助信息

### 2.3 图标规范

#### 图标风格
- 线性图标为主，避免面性图标
- 线条粗细：4rpx
- 圆角处理：4rpx圆角
- 颜色：瓦当黛（#3D3D3D）为主，青砖灰（#9E9E9E）为次

#### 常用图标尺寸
- 导航图标：48x48rpx
- 按钮图标：40x40rpx
- 列表图标：32x32rpx

## 3. 布局结构

### 3.1 页面结构

```
┌─────────────────┐
│ 原生 NavigationBar │
├─────────────────┤
│    内容区域     │
│  (支持 ScrollView) │
│                 │
├─────────────────┤
│   原生 TabBar    │
└─────────────────┘
```

### 3.2 间距规范
- 页面边距：32rpx
- 元素间距：16rpx倍数递增
- 行间距：文字大小的1.5倍
- 模块间距：48rpx

### 3.3 uvue 样式限制规范 (重要)
由于 `.uvue` 在 App 端使用原生渲染引擎，编写样式时必须遵循以下限制：
1. **默认布局**：所有组件默认 `display: flex` 且 `flex-direction: column`。
2. **文本嵌套**：文字必须包裹在 `<text>` 标签内，绝对不允许直接写在 `<view>` 中。
3. **不支持的 CSS**：
   - 移除 `float`、`clear`。
   - 不支持 `display: block/inline/inline-block`，仅支持 `flex` 和 `none`。
   - 不支持复杂的层级选择器（如 `.parent .child`），强制使用单类名（Class）选择器（如 `.parent-child`）。
4. **单位**：强制使用 `rpx` 或 `px`，不支持 `em`、`rem`、`vh`、`vw` 等 Web 特有单位。

## 4. 交互设计

### 4.1 导航交互
- **底部导航 (TabBar)**：使用小程序原生 `tabBar`，3-5个主要功能入口，不要自定义实现以保证性能和点击体验。
- **顶部导航 (NavigationBar)**：尽量使用原生导航栏；如需沉浸式设计（自定义导航栏），需计算状态栏高度和胶囊按钮安全区（通过 `uni.getSystemInfoSync()` 和 `uni.getMenuButtonBoundingClientRect()`）。
- **页面切换**：小程序原生切换动画。

### 4.2 按钮交互
- **普通按钮**：梁木褐背景，素墙白文字
- **主要按钮**：窗棂朱背景，素墙白文字
- **次要按钮**：青砖灰边框，瓦当黛文字
- **悬停效果**：轻微放大（1.05倍）+ 阴影

### 4.3 加载状态
- **骨架屏**：素墙白背景，青砖灰线条
- **加载动画**：窗棂朱色旋转动画
- **错误提示**：窗棂朱背景，白色文字

## 5. 内容展示规范

### 5.1 建筑信息展示

```
┌─────────────────┐
│    建筑图片     │
├─────────────────┤
│    建筑名称     │
│    建筑年代     │
│    地理位置     │
├─────────────────┤
│    文化介绍     │
│    历史价值     │
│    保护现状     │
└─────────────────┘
```

### 5.2 图片展示
- 主图尺寸：宽度100%，高度自适应
- 缩略图：80x80px，圆角8px
- 图片间距：16px
- 图片质量：WebP格式，压缩率80%

## 6. 特殊场景处理

### 6.1 深色模式
- 背景：青砖灰（#9E9E9E）→ 梁木褐（#7A5C48）
- 文字：瓦当黛（#3D3D3D）→ 素墙白（#F8F5F0）
- 避免使用纯黑背景

### 6.2 多语言支持
- 中文优先：确保中文显示效果最佳
- 英文适配：使用对应的英文术语
- 字体回退：确保多语言环境下的显示一致性

## 7. 性能优化

### 7.1 资源优化
- 图片：使用WebP格式，适当压缩
- 字体：仅加载必要的字体子集
- 图标：使用SVG图标，减少资源体积

### 7.2 加载策略
- 懒加载：非首屏内容延迟加载
- 预加载：用户可能访问的内容提前加载
- 缓存策略：合理设置HTTP缓存

## 8. 测试规范

### 8.1 兼容性测试
- 设备：主流手机型号覆盖
- 系统：iOS 12+，Android 6.0+
- 屏幕：全面屏、刘海屏适配

### 8.2 可访问性测试
- 对比度：确保文字可读性
- 触控区域：最小44x44px
- 屏幕阅读器：支持无障碍访问

### 8.3 文化准确性测试
- 建筑细节：确保建筑元素准确呈现
- 文字内容：专业术语准确无误
- 文化敏感性：避免文化误读

## 9. 迭代更新

### 9.1 版本控制
- 主版本：重大功能更新
- 次版本：新增功能或优化
- 修订版本：Bug修复

### 9.2 用户反馈
- 反馈入口：设置页面提供反馈渠道
- 数据分析：收集用户行为数据
- 定期优化：根据反馈持续改进
