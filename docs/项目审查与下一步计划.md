# 中国古建筑智慧学习平台 - 项目审查与下一步计划

## 1. 项目现状审查 (更新于2026年3月31日)

当前项目从基于 Vue 3 + Vite + Tailwind CSS 的 Web 应用向 **uni-app x (uvue + uts)** 架构的迁移已取得阶段性成果。
经过前期的改造，项目**不仅完成了视图层（View Layer）重构，还完成了工程配置清理与逻辑层 (uts) 架构的基础搭建**。

### ✅ 1.1 已完成的迁移工作
1. **文件与标签体系**：页面文件已全部转换为 `.uvue` 格式，全面使用原生组件 (`view`, `text`, `scroll-view`, `map` 等)。
2. **路由与样式重构**：已使用原生的 `pages.json` 进行路由和底部 TabBar 配置；样式已重写为纯 Flex 布局，适应 uni-app x 原生渲染限制。
3. **工程清理与规范化**：
   - 彻底清除了 `tailwind.config.js`, `postcss.config.js`, `style.css` 等 Web 遗留文件。
   - `package.json` 已完成依赖净化，移除了多余的 Web 样式库依赖。
   - 静态资源已规范化统一收敛至 `static/` 目录。
4. **逻辑层 (uts) 架构搭建与解耦**：
   - 已建立标准化的 `services/` (API请求层)、`types/` (类型定义)、`store/` (状态管理) 等目录。
   - 在 `types/` 下建立了完善的业务数据结构（如 `User`, `Achievement`, `KnowledgeNode` 等），实现了强类型 `uts`。
   - 数据与视图成功解耦，各页面的数据不再硬编码在模板中，而是通过 `services/` 下的 Mock 服务提供，由页面通过生命周期调用。
   - 已在 `store/user.uts` 中基于 Vue 3 `reactive` 实现了符合 uni-app x 规范的轻量级全局状态管理。

### 🚨 1.2 当前遗留问题（待优化清单）
1. **真实接口未对接**：目前 `services/` 目录下的逻辑全部基于 `setTimeout` 和 Mock 数据，尚未与真实后端 API 对接。
2. **地图服务尚未调用真实定位**：虽然使用了原生 `<map>` 组件，但数据为静态 Mock，尚未结合 uni-app 原生定位 API (`uni.getLocation`) 获取真实经纬度和动态 Markers。
3. **AI 对话未连接大模型**：`services/ai.uts` 中的回复依然是本地生成的占位文本，缺乏真实的流式对话交互。
4. **组件复用度低**：尚缺乏全局复用的 UI 组件（如通用的导航栏、空状态提示等），`components/` 目录尚未发挥作用。

---

## 2. 下一步开发计划

随着工程清理与架构基础的完成，下一步的核心目标是**从 Mock 数据转向真实业务，完成核心 API 对接与原生功能深化**。

### 第一阶段：核心业务逻辑与 API 对接 (进行中)
* **地图服务深化**：接入真实的定位服务 API (`uni.getLocation`)，并根据实时经纬度向后端请求附近的古建筑标记点数据。
* **AI 导学模块对接**：在 `services/ai.uts` 中实现与后端 AI 大模型（或相关云服务）的真实接口通信，处理流式回复 (SSE) 等复杂场景。
* **全局状态落地**：将 `store/user.uts` 的登录状态、学习进度与真实的鉴权流程（如 `uni.login` 和 Token 管理）结合起来。

### 第二阶段：UI 细节优化与组件化抽取
* **基础组件封装**：在 `components/` 目录下提取高频复用的 UI 组件（例如通用的 `CustomButton`, `DataLoading`, `EmptyState` 等）。
* **原生体验提升**：优化长列表性能，将部分页面的 `scroll-view` 替换为更高效的 `list-view`（App 端性能更佳）；加入页面切换动画与交互反馈。
* **多端适配微调**：虽然基于 uni-app x，但仍需确保在 Android/iOS App 端和微信小程序端的样式和交互表现一致。

### 第三阶段：测试与发布准备
* **真机调试**：连接 Android / iOS 真机进行性能和内存压测，确保没有任何 Web 时代的 DOM API 残留导致 App 端崩溃。
* **小程序端预览**：在微信开发者工具中预览并处理小程序特有的权限与安全域名配置。

---

## 3. 结论

项目的视图层 (uvue) 改造与逻辑层 (uts) 架构搭建已经基本完成，代码结构清晰、类型严谨。目前的当务之急是**对接真实数据与原生能力**，通过实现地图定位、AI 对话和真实状态流转，让项目具备完整的 uni-app x 原生级别开发体验和性能表现。
